# FlowForge Customization PRD

## Product Requirements Document
**Version:** 1.0  
**Date:** January 2026  
**Author:** FlowForge Team  

---

## 1. Overview

This document outlines all customizations required to fork Auto-Claude and create **FlowForge** - a branded, enhanced version with additional Claude usage tracking features.

### 1.1 Goals
- Rebrand Auto-Claude as "FlowForge" with custom icons and logos
- Add a dedicated Claude Usage tracking view
- Add quick usage indicators in the left sidebar
- Maintain full compatibility with upstream Auto-Claude updates

### 1.2 Non-Goals
- Changing core Auto-Claude functionality
- Modifying the agent/backend architecture
- Adding features unrelated to usage tracking

## 6. Technical Notes

### 6.1 Electron API Extensions

The following IPC channels are used:

| Channel | Direction | Purpose |
|---------|-----------|---------|
| `claude:get-profiles` | Main â† Renderer | Get all Claude profiles |
| `claude:switch-profile` | Main â† Renderer | Switch active profile |
| `claude:request-usage-update` | Main â† Renderer | Fetch current usage |
| `claude:usage-updated` | Main â†’ Renderer | Usage data broadcast |

### 6.2 Existing Infrastructure

Leverage existing code:
- `src/main/claude-profile/` - Profile management
- `src/main/usage-parser.ts` - Usage data parsing
- `src/renderer/shared/components/ui/` - UI components

### 6.3 Build Considerations

For production builds with `externalizeDeps: false`:
- All dependencies bundled into main.js
- No node_modules in packaged app
- Electron version: 39.x recommended
